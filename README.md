# Black Building Academy — HRMS Mobile App

A production-style **Flutter** mobile app for the Black Building Academy
tuition center HRMS. Connects directly to **ERPNext / Frappe** via REST APIs
and a small set of custom whitelisted methods on the server.

Designed for ~50–100 employees with two roles:

- **Employee** — dashboard, attendance, check-in/out, leaves, tasks, salary
  slips, holidays.
- **HR Manager** — analytics dashboard, attendance management, leave
  approvals, task assignment, payroll overview, employee directory.

Tech: Flutter 3.22+, Riverpod 2, Dio 5, go_router 14, flutter_secure_storage,
table_calendar, fl_chart, geolocator. Dark/gold material 3 design system.

## 1. Prerequisites

- Flutter SDK **3.22+** (Dart **3.4+**) — `flutter doctor` should be green.
- Android Studio / Xcode for device builds.
- An **ERPNext** site running the `bb_acadamy_admin` app with the new
  `bb_acadamy_admin/api/mobile.py` module (added in this PR).

## 2. Get the Flutter project ready

The repo ships only the Dart source — platform folders are generated by the
Flutter CLI to keep them in sync with your installed SDK.

```bash
# 1. From the project root
flutter create . --platforms=android,ios --project-name bb_academy_hrms

# 2. Install dependencies
flutter pub get

# 3. Copy and edit the .env file
cp .env.example .env
# → set API_BASE_URL to your ERPNext URL (no trailing slash)

# 4. Add the Inter font files to assets/fonts/
#    Inter-Regular.ttf, Inter-Medium.ttf, Inter-SemiBold.ttf, Inter-Bold.ttf
#    (https://fonts.google.com/specimen/Inter)

# 5. Run
flutter run
```

### Android permissions

Add to `android/app/src/main/AndroidManifest.xml`:

```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION" />
```

### iOS permissions

In `ios/Runner/Info.plist`:

```xml
<key>NSLocationWhenInUseUsageDescription</key>
<string>Used to record check-in location.</string>
```

## 3. Backend setup (Frappe)

Pull the latest `bb_acadamy_admin` app into your bench and migrate:

```bash
bench --site <site-name> migrate
bench restart
```

Verify the API endpoints by opening:

```
GET  https://<site>/api/method/bb_acadamy_admin.api.mobile.me
```

after authenticating in the same browser session.

### Required user setup

1. Each staff member must have an **Employee** record with `user_id` set to a
   real **User** with the `Employee` role.
2. HR users must have `HR Manager` or `HR User` role.
3. The Employee should have `holiday_list`, `custom_expected_in_time`,
   `custom_expected_out_time`, and `custom_grace_minutes` set so attendance
   late/early calculations work.

Each feature follows the same shape:

```
features/<feature>/
├── data/
│   ├── models/<name>.dart        # Plain immutable Dart models
│   └── <feature>_repository.dart # Talks to DioClient, returns models
└── presentation/
    ├── controllers/              # StateNotifiers / providers (when stateful)
    ├── screens/                  # ConsumerWidgets bound to providers
    └── widgets/                  # Per-feature UI fragments
```

State is managed with **Riverpod 2** providers; `FutureProvider.autoDispose`
is the default for read screens, with `StateNotifierProvider` for flows that
need transitions (e.g. auth, check-in).

## 5. Authentication flow

1. User submits username + password on `LoginScreen`.
2. App POSTs to `/api/method/login` (`x-www-form-urlencoded`).
3. Frappe responds with a `Set-Cookie: sid=…` header.
4. The `sid` is extracted and persisted via `flutter_secure_storage`.
5. `AuthInterceptor` injects the `sid` cookie (or an
   `Authorization: token <key>:<secret>` header if the user later switches to
   token auth) on every subsequent request.
6. The router watches `authControllerProvider` and routes the user to the
   `Employee` shell or the `HR` shell based on their roles.

## 6. ERPNext / Frappe API map

| Feature        | Endpoint                                                                 | Method | Notes                                                                  |
|----------------|--------------------------------------------------------------------------|--------|------------------------------------------------------------------------|
| Login          | `/api/method/login`                                                      | POST   | Form-encoded `usr`, `pwd` body. Sets `sid` cookie.                     |
| Logout         | `/api/method/logout`                                                     | GET    |                                                                        |
| Current user   | `/api/method/bb_acadamy_admin.api.mobile.me`                             | GET    | Custom — merges User + Employee + roles.                               |
| Emp dashboard  | `/api/method/bb_acadamy_admin.api.mobile.employee_dashboard`             | GET    | Custom — single round-trip for home screen.                            |
| HR dashboard   | `/api/method/bb_acadamy_admin.api.mobile.hr_dashboard`                   | GET    | Custom — HR aggregates + recent activity.                              |
| Check in       | `/api/method/bb_acadamy_admin.api.mobile.check_in`                       | POST   | Body: `latitude`, `longitude`, `time`. Creates Employee Checkin (IN).  |
| Check out      | `/api/method/bb_acadamy_admin.api.mobile.check_out`                      | POST   | Same shape as `check_in` with log_type=OUT.                            |
| Today status   | `/api/method/bb_acadamy_admin.api.mobile.today_checkin_status`           | GET    | Used to render dashboard chip.                                         |
| Monthly attn.  | `/api/method/bb_acadamy_admin.api.mobile.monthly_attendance`             | GET    | Query: `year`, `month`. Returns days[] + summary.                      |
| Leave balance  | `/api/method/bb_acadamy_admin.api.mobile.leave_balance`                  | GET    | Per-leave-type balances.                                               |
| Apply leave    | `/api/method/bb_acadamy_admin.api.mobile.apply_leave`                    | POST   | `leave_type`, `from_date`, `to_date`, `description`, `half_day`.       |
| Approve leave  | `/api/method/bb_acadamy_admin.api.mobile.approve_leave`                  | POST   | HR-only. `name`, `status` (Approved/Rejected), `note`.                 |
| My salary slips| `/api/method/bb_acadamy_admin.api.mobile.my_salary_slips`                | GET    | Submitted slips for the logged-in employee.                            |
| Salary detail  | `/api/method/bb_acadamy_admin.api.mobile.salary_slip_detail`             | GET    | `name=…`. Includes earnings & deductions breakdown.                    |
| Salary PDF     | `/api/method/frappe.utils.print_format.download_pdf`                     | GET    | Standard Frappe printer. Opens in browser via `url_launcher`.          |
| Leave list     | `/api/resource/Leave Application`                                        | GET    | Standard CRUD. Used for history + approval queue.                      |
| Leave types    | `/api/resource/Leave Type`                                               | GET    |                                                                        |
| Tasks (mine)   | `/api/resource/Task`                                                     | GET    | Filtered by `_assign like %username%`.                                 |
| Task update    | `/api/resource/Task/<name>`                                              | PUT    | progress + status update.                                              |
| Comment        | `/api/method/frappe.desk.form.utils.add_comment`                         | POST   | Used for task comments.                                                |
| Assign task    | `/api/method/frappe.desk.form.assign_to.add`                             | POST   | HR-only — assigns a Task to a User.                                    |
| Employees      | `/api/resource/Employee`                                                 | GET    | HR directory + assignee picker.                                        |
| Holiday list   | `/api/resource/Holiday List`                                             | GET    |                                                                        |
| Attendance     | `/api/resource/Attendance`                                               | GET    | HR daily register.                                                     |

## 7. Reusable widgets

- **`PrimaryButton`** — full-width pill button with dark / gold / danger /
  success variants and built-in loading state.
- **`AppTextField`** — labeled, themed text input.
- **`StatCard`** — dashboard metric tile (icon + label + value).
- **`StatusBadge`** — pill chip with colour from status string
  (`StatusBadge.forStatus("Approved")`).
- **`EmptyState`** / **`LoadingView`** / **`ErrorView`** — consistent
  placeholder treatments across every list screen.
- **`SectionHeader`** — section title row, optional trailing action.

Set `APP_ENV=prod` and `API_BASE_URL=https://erp.blackbuilding.com` in

## 10. Testing tips

- During dev, use **Frappe API Token** instead of session login by setting
  `API_KEY` and `API_SECRET` in `.env` — the interceptor automatically
  switches to header auth.
- The `dio_client.dart` enables `pretty_dio_logger` in debug mode so every
  request is printed to the Flutter console.
- For local Frappe + Android emulator, use `http://10.0.2.2:8000` as the
  base URL.
